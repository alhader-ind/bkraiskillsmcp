# Bootstrap Protocol: Astro Hybrid Site

**Use Case:** The user requests a high-performance content-driven site, a static site generator (SSG), or an Astro web application requiring UI framework integrations.

> [!CRITICAL] INTERACTIVE CLI WARNING
> The `create-astro` command is highly interactive. You MUST use strict flags to bypass the prompt blocking, followed by immediate non-interactive integration additions.

## Step 1: Force Non-Interactive Scaffolding
Execute the official Astro creator using specific flags to ensure it skips all prompts.

**Execution Command:**
```bash
npx -y create-astro@latest . --template basics --install --no-git --typescript strict --yes
```

## Step 2: Add Official Integrations Non-Interactively
Post-initialization, the AI MUST immediately add essential integrations (React and Tailwind) before the agent does any manual configuration. Astro CLI provides `astro add` which we must run with `--yes`.

**Execution Command:**
```bash
npx -y astro add react tailwind --yes
```

## Step 3: Verify and Inject Global Styles
Astro and Tailwind config will be auto-generated by the previous step, but global CSS must form the baseline.

**Create `src/styles/global.css`**
```css
@tailwind base;
@tailwind components;
@tailwind utilities;

@layer base {
  body {
    @apply bg-slate-950 text-slate-50 font-sans antialiased;
  }
}
```

## Step 4: Update the Base Layout
Wire the global styles into the main layout component.

**Update `src/layouts/Layout.astro`**
```astro
---
import '../styles/global.css';

interface Props {
	title: string;
}

const { title } = Astro.props;
---

<!doctype html>
<html lang="en">
	<head>
		<meta charset="UTF-8" />
		<meta name="description" content="Astro Bootstrapped Site" />
		<meta name="viewport" content="width=device-width" />
		<link rel="icon" type="image/svg+xml" href="/favicon.svg" />
		<meta name="generator" content={Astro.generator} />
		<title>{title}</title>
	</head>
	<body>
		<slot />
	</body>
</html>
```

## Step 5: Test React Integration Boundary
Create a React component to verify the island architecture.

**Create `src/components/AstroReactIsland.tsx`**
```tsx
import React, { useState } from 'react';

export default function AstroReactIsland() {
  const [count, setCount] = useState(0);

  return (
    <div className="p-6 bg-slate-900 border border-slate-700 rounded-xl max-w-sm mt-6">
      <h3 className="text-xl font-bold text-sky-400 mb-2">React Island Active</h3>
      <p className="text-slate-400 mb-4 text-sm">
        This component is fully interactive, hydrated via Astro client directives.
      </p>
      <button 
        onClick={() => setCount(c => c + 1)}
        className="px-4 py-2 bg-sky-600 hover:bg-sky-500 rounded font-medium transition-colors"
      >
        Interactions: {count}
      </button>
    </div>
  );
}
```

## Step 6: Scaffold the Main Page
Update the index page to utilize the standard Astro structural elements and the React Island.

**Update `src/pages/index.astro`**
```astro
---
import Layout from '../layouts/Layout.astro';
import AstroReactIsland from '../components/AstroReactIsland.tsx';
---

<Layout title="Astro Hybrid Activated">
	<main class="min-h-screen flex flex-col items-center justify-center p-8 text-center">
		<h1 class="text-5xl font-black tracking-tight text-transparent bg-clip-text bg-gradient-to-r from-purple-400 to-sky-400">
			Astro Protocol
		</h1>
		<p class="mt-4 text-slate-400 max-w-lg text-lg">
			Your hybrid site has been successfully bootstrapped with strict TypeScript, Tailwind CSS, and React Integrations.
		</p>
		
		<AstroReactIsland client:load />
	</main>
</Layout>
```

## Step 7: Verify & Yield
Execute `compile_applet` and verify `package.json` scripts (`npm run dev`). Astro dev runs fast, ensure there are no compilation errors regarding TSX or Tailwind imports.
