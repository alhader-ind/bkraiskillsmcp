# Bootstrap Protocol: React Native + Expo Router

**Use Case:** The user requests a mobile application using React Native.

> [!CRITICAL] NATIVE FILES WARNING
> You MUST NEVER attempt to manually configure iOS (`/ios`) or Android (`/android`) native files. Always use Expo to abstract native complexity.

## Step 1: Force Non-Interactive Scaffolding
Execute the Expo CLI using precise flags to bypass human prompts and enforce a blank TypeScript template. This stops the AI from stalling on template selection.

**Execution Command:**
```bash
npx create-expo-app@latest . --yes --template expo-template-blank-typescript
```

## Step 2: Ensure Strict Expo Router Compliance
If the user requests multiple screens or nested navigation, you MUST configure the modern `expo-router` paradigm instead of legacy React Navigation.

**Execution Command:**
```bash
npx expo install expo-router react-native-safe-area-context react-native-screens expo-linking expo-constants expo-status-bar
```

## Step 3: Configuration Injection
Adjust `app.json` and `package.json` to firmly point the app at the `expo-router`.

**Modify `package.json` (Snippet)**
Ensure the `main` entry point points to expo router, not App.js.
```json
{
  "main": "expo-router/entry"
}
```

**Modify `app.json`**
```json
{
  "expo": {
    "name": "ai-expo-app",
    "slug": "ai-expo-app",
    "scheme": "my-ai-app",
    "plugins": ["expo-router"],
    "experiments": {
      "typedRoutes": true
    }
  }
}
```

## Step 4: Routing Scaffold
Remove the default `App.tsx` generated by the blank template. Set up root layout boundaries for Expo Router inside the `/app` folder.

**`app/_layout.tsx`**
```tsx
import { Stack } from "expo-router";

export default function RootLayout() {
  return (
    <Stack>
      <Stack.Screen name="index" options={{ title: "Home" }} />
    </Stack>
  );
}
```

**`app/index.tsx`**
```tsx
import { View, Text, StyleSheet } from 'react-native';
import { StatusBar } from 'expo-status-bar';

export default function HomeScreen() {
  return (
    <View style={styles.container}>
      <Text style={styles.title}>Expo Protocol Active</Text>
      <Text style={styles.subtitle}>Native folders are safely abstracted.</Text>
      <StatusBar style="auto" />
    </View>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#0f172a',
    alignItems: 'center',
    justifyContent: 'center',
    padding: 20,
  },
  title: {
    color: '#38bdf8',
    fontSize: 24,
    fontWeight: 'bold',
    marginBottom: 8,
  },
  subtitle: {
    color: '#94a3b8',
    fontSize: 16,
    textAlign: 'center',
  }
});
```

## Step 5: Verify & Yield
Execute `compile_applet` equivalents (or run standard Expo TS checks). Yield success to the user so they can begin providing native feature requests.
