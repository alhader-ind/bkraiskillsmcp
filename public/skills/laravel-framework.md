# Laravel Framework Guidelines

Laravel is a web application framework with expressive, elegant syntax. It provides a structured approach to building applications with features like expressive routing, powerful dependency injection, and a robust ORM.

## 1. Project Structure (Laravel 11+)

- **app/**: Contains the core code of your application (Models, Providers).
- **bootstrap/app.php**: The main entry point for configuring the application, including middleware and routing.
- **config/**: Application configuration files.
- **database/**: Database migrations, model factories, and seeds.
- **public/**: The document root for the application, contains the index.php file and assets.
- **resources/**: Contains your views as well as raw, un-compiled assets such as CSS or JavaScript.
- **routes/**: All route definitions for your application (web.php, console.php). APIs are handled via routes/api.php if installed.
- **storage/**: Compiled Blade templates, file-based sessions, file caches, and other files generated by the framework.

## 2. Core Best Practices

- **Minimalist Configuration**: In newer Laravel versions, many configuration files are hidden by default. Use `.env` or publish only needed configs.
- **Eloquent ORM**: Always use Eloquent for database interactions to maintain readability and leverage built-in security features like SQL injection prevention.
- **Service Layer**: For complex logic, extract code from Controllers into Service classes.
- **Type Hinting**: Utilize PHP 8.2+ features like readonly properties and strict type hinting in Models and Controllers.
- **Vite Integration**: Use Vite for bundling frontend assets.

## 3. Frontend Strategies

- **Inertia.js**: The recommended way to build modern single-page apps using Vue or React within the Laravel ecosystem.
- **Laravel Livewire**: An excellent alternative for building dynamic interfaces without leaving the comfort of Laravel.
- **Blade Components**: Use anonymous or class-based Blade components for reusable UI elements.

## 4. Real-time Features

- **Laravel Reverb**: Use the first-party WebSocket server for building real-time applications with ease.

## 5. Example Controller

```php
<?php

namespace App\Http\Controllers;

use App\Models\Post;
use Illuminate\Http\Request;
use Inertia\Inertia;

class PostController extends Controller
{
    public function index()
    {
        return Inertia::render('Posts/Index', [
            'posts' => Post::latest()->get()
        ]);
    }

    public function store(Request $request)
    {
        $validated = $request->validate([
            'title' => 'required|string|max:255',
            'content' => 'required|string',
        ]);

        Post::create($validated);

        return redirect()->route('posts.index');
    }
}
```
